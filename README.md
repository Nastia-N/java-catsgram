# Catsgram
REST API для социальной сети, где пользователи могут делиться фотографиями своих котов.
Приложение построено на Spring Boot и предоставляет функционал для управления пользователями, публикациями и изображениями.

## Основные возможности
Управление пользователями: регистрация, обновление профиля, просмотр списка пользователей  
Публикации: создание, редактирование и просмотр постов  
Изображения: загрузка нескольких изображений к постам, скачивание файлов  
Валидация данных: централизованная обработка ошибок и проверка входных данных  
Хранение файлов: сохранение изображений в файловой системе с организацией по папкам пользователей и постов

## Особенности реализации
#### Хранение изображений
Изображения сохраняются в файловой системе по пути: {image-directory}/{userId}/{postId}/{timestamp}.{extension}  
Метаданные об изображениях хранятся в базе данных  
Поддерживается загрузка нескольких файлов одновременно

#### Валидация
Проверка уникальности email пользователей  
Проверка существования связанных сущностей (автор поста, пост для изображения)  
Проверка обязательных полей

## Технологический стек
Backend: Java 21, Spring Boot 3
Фреймворки: Spring MVC, Spring JDBC
Базы данных: PostgreSQL
Инструменты: Jakarta Validation, Lombok, Maven, Gson
Система контроля версий: Git

## Эндпоинты

#### Домашняя страница (/home):
GET    /home                    - приветственная страница приложения

#### Пользователи (/users):
GET    /users                    - получить всех пользователей  
GET    /users/{id}              - получить пользователя по ID  
POST   /users                    - создать нового пользователя  
PUT    /users/{id}              - обновить данные пользователя

#### Публикации (/posts):
GET    /posts/{id}              - получить публикацию по ID  
POST   /posts                    - создать новую публикацию  
PUT    /posts/{id}              - обновить описание публикации

#### Изображения (/posts/{postId}/images):
GET    /posts/{postId}/images   - получить все изображения публикации  
POST   /posts/{postId}/images   - загрузить изображения к публикации (multipart/form-data)  
GET    /posts/{postId}/images/{imageId} - скачать конкретное изображение
